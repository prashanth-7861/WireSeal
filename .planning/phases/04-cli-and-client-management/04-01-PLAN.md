--- init ---
@cli.command()
@click.option("--subnet", default="10.0.0.0/24", help="VPN subnet (RFC 1918)")
@click.option("--port", default=51820, type=int, help="WireGuard listen port")
@click.option("--duckdns-domain", default=None, help="DuckDNS subdomain (optional)")
def init(subnet, port, duckdns_domain):
  1. Collect vault passphrase twice (passphrase + confirm) via
     click.prompt(hide_input=True, confirmation_prompt=True).
     Enforce minimum 12 characters — raise click.ClickException if shorter.
  2. Import Vault from security.vault; create a new vault at the default path
     (~/.wg-automate/vault.enc). If vault already exists, abort with message
     "Vault already exists. Run verify to check integrity."
  3. Import get_adapter from platform.detect; call adapter.check_privileges()
     (raises PrivilegeError if not root/admin).
  4. Import generate_keypair from core.keygen; generate server keypair.
     Store private key (SecretBytes) and public key in vault state.
  5. Import IPPool from core.ip_pool; allocate server IP (.1) and store
     subnet config in vault state.
  6. Import ConfigBuilder from core.config_builder; build server wg0.conf
     using the server keypair and subnet. Write via adapter.deploy_config().
  7. Call adapter.install_wireguard(), adapter.configure_firewall(),
     adapter.start_service() in sequence.
  8. If duckdns_domain is provided: prompt for DuckDNS token via
     click.prompt(hide_input=True). Store token as SecretBytes in vault state.
  9. Import AuditLog from security.audit; log action="init" with metadata
     (subnet, port, platform). No passphrase or key material in log entry.
  10. Print success summary: subnet, server IP, public key, port.

--- status ---
@cli.command()
def status():
  1. Collect passphrase via click.prompt(hide_input=True).
  2. Open vault to confirm it is unlockable (proves auth). Close immediately.
     The vault open here is just an integrity check — status data comes from wg show.
  3. Run subprocess: ["wg", "show"] with capture_output=True, text=True.
     If wg show returns non-zero, print "WireGuard interface not running" and exit 1.
  4. Parse wg show output:
     - Extract interface name, public key (display truncated: first 8 chars + "..."),
       listen-port, and fwmark.
     - For each peer: extract public key (truncated), allowed IPs, latest handshake,
       transfer (rx/tx bytes).
     - NEVER display private keys or PSKs. The wg show output does not include
       private keys — but confirm by asserting "PrivateKey" does not appear in output.
  5. Print formatted table of peers. If no peers: print "No clients configured."
  6. Log action="status" to AuditLog (no peer key material in log).

--- verify ---
@cli.command()
def verify():
  1. Collect passphrase via click.prompt(hide_input=True).
  2. Open vault via Vault context manager.
  3. Import check_all from security.integrity (or equivalent batch verifier).
     Pass deployed config file paths from vault state.
  4. For each config path in vault state:
     - Compute SHA-256 of file on disk.
     - Compare to stored hash in vault.
     - If mismatch: print "[TAMPERED] /path/to/config" in red (click.style).
     - If match: print "[OK] /path/to/config" in green.
  5. If any mismatches found: exit with code 1 and print summary
     "ALERT: {N} config file(s) have been tampered with."
  6. If all OK: print "All config files verified. No tampering detected."
  7. Log action="verify", result="ok|tampered", tampered_count=N.
  This implements CLI-03.

--- lock ---
@cli.command()
def lock():
  1. This command explicitly does NOT open the vault (no passphrase needed).
  2. Scan for any in-memory decrypted state artifacts (e.g., temp files under
     ~/.wg-automate/ that are not vault.enc). Remove any found.
  3. Print "Vault locked. All decrypted state wiped."
  4. Log action="lock" to AuditLog if audit log is accessible without vault.
     If audit log is not accessible: skip logging (lock must never fail).
  5. sys.exit(0).
  This implements CLI-04. The security guarantee: decrypted state only ever
  exists in Python memory (Vault context manager's finally block wipes it).
  lock() is the manual signal that the session is over.

--- change-passphrase ---
@cli.command("change-passphrase")
def change_passphrase():
  1. Collect current passphrase via click.prompt("Current passphrase", hide_input=True).
  2. Collect new passphrase via click.prompt("New passphrase", hide_input=True,
     confirmation_prompt=True). Enforce minimum 12 characters.
  3. Import Vault from security.vault; call vault.change_passphrase(old, new).
     This decrypts with old, re-encrypts with new, atomically writes.
  4. Wipe both passphrase variables from memory after use.
  5. Print "Passphrase changed successfully."
  6. Log action="change-passphrase" (no passphrase material in log).
  This implements VAULT-07 at the CLI surface.

SECURITY INVARIANT for all commands: Never store passphrase beyond the
duration of the vault open call. Wipe via secrets_wipe.wipe_string() or
bytearray zeroing after use. Raise click.ClickException (not bare exceptions)
so Click formats errors cleanly without Python tracebacks leaking locals.
