**tests/security/test_secret_types.py** -- Tests for `wg_automate.security.secret_types.SecretBytes`
and `wg_automate.security.secrets_wipe.wipe_bytes`:
- `test_repr_does_not_expose_secret`: `repr(sb)` must not contain the raw bytes value.
- `test_str_does_not_expose_secret`: `str(sb)` must not contain the raw bytes value.
- `test_eq_compares_equal_buffers`: Two SecretBytes wrapping identical bytearrays are equal.
- `test_eq_does_not_expose_via_timing`: Simply assert equal/not-equal works (no timing assertion
  needed -- just ensure the method functions correctly).
- `test_hash_raises_or_returns_constant`: Either __hash__ raises TypeError or returns a constant
  value (never the content hash). Assert it does not reveal content.
- `test_getstate_raises_or_returns_safe`: pickle.dumps(sb) either raises or produces output that
  does not contain the raw secret bytes. Use `pytest.raises(Exception)` or check pickled bytes.
- `test_wipe_zeroes_buffer`: Create a bytearray, call `wipe_bytes(buf)`, assert all bytes are 0
  OR that the buffer length is 0. Do NOT assert mlock succeeded (see Research anti-pattern:
  never assert mlock success -- it is best-effort).
- `test_secretbytes_created_and_wiped_without_error`: Construct `SecretBytes(bytearray(b"secret"))`,
  call `sb.wipe()` (or equivalent wipe method discovered by reading the source), assert no
  exception raised. This catches PyInstaller ctypes regressions (Research open question 1).

**tests/security/test_vault.py** -- Tests for `wg_automate.security.vault.Vault`:
Use the `vault_path`, `passphrase`, `wrong_passphrase`, `initial_vault_state` fixtures from
conftest.py. Read vault.py first to identify the exact API (Vault.create, Vault(), vault.open,
context manager pattern, change_passphrase).
- `test_vault_round_trip`: Create vault, reopen with same passphrase, assert schema_version == 1.
- `test_wrong_passphrase_raises`: Open with wrong_passphrase, assert VaultUnlockError raised.
- `test_tampered_ciphertext_raises`: After creating, flip byte at index 51 (past header),
  assert VaultUnlockError raised when reopening.
- `test_bad_magic_raises_tampered_error`: Overwrite first 4 bytes with b"XXXX", assert
  VaultTamperedError raised.
- `test_atomic_write_leaves_no_partial_on_fsync_crash`: Patch `os.fsync` with side_effect OSError,
  call Vault.create, assert vault_path does not exist and no .tmp_ files remain in tmp_path.
  (Applies if vault uses atomic_write internally -- if not, skip and cover in test_atomic.py.)
- `test_passphrase_change_preserves_state`: Create vault, call change_passphrase with new
  passphrase, reopen with new passphrase, assert data intact; assert old passphrase raises
  VaultUnlockError.

**tests/security/test_atomic.py** -- Tests for `wg_automate.security.atomic.atomic_write`:
Read atomic.py to confirm function signature and tmp file naming pattern.
- `test_atomic_write_success`: Write b"hello" to dest, assert dest.read_bytes() == b"hello",
  assert no .tmp_ files remain.
- `test_atomic_write_no_partial_on_fsync_crash`: Patch os.fsync side_effect=OSError("disk full"),
  call atomic_write, assert pytest.raises(OSError), assert dest does not exist, assert no
  leftover tmp files matching `.tmp_wga_*` (or whatever prefix the implementation uses -- read
  atomic.py first to confirm prefix).
- `test_atomic_write_overwrites_existing`: Write "v1" then write "v2" to same dest, assert "v2".

**tests/security/test_permissions.py** -- Tests for `wg_automate.security.permissions`:
Read permissions.py to identify the function names (likely `set_file_permissions(path, platform)`
or similar). Use `mocker` fixture from pytest-mock.
- `test_unix_permissions_calls_chmod_600`: Mock os.chmod, call the permissions function with a
  tmp file on simulated Linux/macOS, assert os.chmod called with mode 0o600.
- `test_windows_delegates_to_icacls`: Mock subprocess.run (or pywin32 if used), call permissions
  function with simulated Windows platform, assert icacls command was invoked (or the Windows
  ACL setter was called). Never call real icacls.
- `test_unix_does_not_call_icacls_on_linux`: Assert subprocess.run was NOT called when platform
  is linux.

**tests/security/test_integrity.py** -- Tests for `wg_automate.security.integrity`:
Read integrity.py to identify `compute_config_hash` and `verify_config_integrity` signatures.
- `test_compute_hash_is_sha256_hex`: Write a known file, compute hash, assert it is a 64-char
  hex string and matches `hashlib.sha256(content).hexdigest()`.
- `test_verify_returns_true_for_unchanged_file`: Compute hash, verify immediately, assert True.
- `test_verify_returns_false_after_tampering`: Compute hash, append a line, verify, assert False
  (or assert TamperingDetectedError if that is the design -- check integrity.py).
- `test_verify_returns_false_for_wrong_hash`: Pass a known-wrong hash string, assert False.
