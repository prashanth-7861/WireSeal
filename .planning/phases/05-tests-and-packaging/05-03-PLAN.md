before writing the spec. Base the spec on the Research code example (PKG-02 pattern).

Key constraints from Research:
- `upx=False` -- UPX triggers AV false positives (Research Pitfall 7)
- `console=True` -- CLI tool must never use --windowed
- `strip=False` -- strip can corrupt binaries on some platforms
- For pyinstaller-hooks-contrib >= 2026.0: no manual hiddenimports for cryptography or argon2
  (hooks-contrib handles them automatically). DO NOT add custom hook-cryptography.py.
- Include templates directory if config_builder.py uses filesystem templates:
  `datas=[("src/wg_automate/templates", "wg_automate/templates")]`
  If there is no templates/ directory (templates embedded in Python strings), omit the datas line.
- hiddenimports must list the three platform modules explicitly because they are loaded by
  string name at runtime via the platform detect factory:
  `"wg_automate.platform.linux"`, `"wg_automate.platform.macos"`, `"wg_automate.platform.windows"`

Windows note (Research Pitfall 7): The spec file uses `onefile=True` for all platforms.
Document in README that Windows Defender may flag the binary (AV false positive from PyInstaller
bootloader). The release.yml uses Sigstore for integrity verification, not traditional code signing.
If AV false positives become a blocker, switch Windows to onedir + zip in a future release.

**build.yml** (CI for every push and PR):
This workflow runs tests and builds binaries. It is separate from release.yml so that artifact
verification can run on every push without triggering signing.

```yaml
name: CI

on:
  push:
    branches: ["main", "master"]
  pull_request:
    branches: ["main", "master"]

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: "3.14"
      - name: Install dependencies with hash verification
        run: pip install --require-hashes -r requirements-dev.txt
      - name: Audit dependencies for CVEs
        run: pip-audit -r requirements.txt
      - name: Run unit tests
        run: pytest --tb=short -q -m "not integration"
      - name: Run integration tests
        if: github.event_name == 'push' && github.ref == 'refs/heads/main'
        run: pytest --tb=short -q -m integration
        env:
          DOCKER_BUILDKIT: "1"

  build:
    needs: test
    strategy:
      fail-fast: false
      matrix:
        include:
          - os: ubuntu-latest
            artifact_name: wg-automate-linux-x86_64
          - os: macos-latest
            artifact_name: wg-automate-macos-arm64
          - os: windows-latest
            artifact_name: wg-automate-windows-x86_64.exe
    runs-on: ${{ matrix.os }}
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: "3.14"
      - name: Install dependencies with hash verification
        run: pip install --require-hashes -r requirements-dev.txt
      - name: Build binary
        run: pyinstaller wg-automate.spec
      - name: Stage binary for upload
        shell: bash
        run: |
          mkdir -p dist/release
          find dist -maxdepth 1 \( -name "wg-automate" -o -name "wg-automate.exe" \) | \
            xargs -I{} cp {} dist/release/${{ matrix.artifact_name }}
      - uses: actions/upload-artifact@v4
        with:
          name: ${{ matrix.artifact_name }}
          path: dist/release/${{ matrix.artifact_name }}
```

**release.yml** (on: release published -- triggered when a GitHub Release is created):
Based on the Research code example for PKG-03. Uses Sigstore keyless signing (no stored keys).

```yaml
name: Release

on:
  release:
    types: [published]

jobs:
  build:
    strategy:
      fail-fast: false
      matrix:
        include:
          - os: ubuntu-latest
            artifact_name: wg-automate-linux-x86_64
          - os: macos-latest
            artifact_name: wg-automate-macos-arm64
          - os: windows-latest
            artifact_name: wg-automate-windows-x86_64.exe
    runs-on: ${{ matrix.os }}
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: "3.14"
      - name: Install dependencies with hash verification
        run: pip install --require-hashes -r requirements-dev.txt
      - name: Run tests before release build
        run: pytest --tb=short -q -m "not integration"
      - name: Build binary
        run: pyinstaller wg-automate.spec
      - name: Stage binary
        shell: bash
        run: |
          mkdir -p dist/release
          find dist -maxdepth 1 \( -name "wg-automate" -o -name "wg-automate.exe" \) | \
            xargs -I{} cp {} dist/release/${{ matrix.artifact_name }}
      - uses: actions/upload-artifact@v4
        with:
          name: ${{ matrix.artifact_name }}
          path: dist/release/${{ matrix.artifact_name }}

  sign-and-release:
    needs: build
    runs-on: ubuntu-latest
    permissions:
      contents: write       # Upload to GitHub Release
      id-token: write       # Required for Sigstore keyless OIDC signing
    steps:
      - uses: actions/download-artifact@v4
        with:
          path: dist/release
          merge-multiple: true
      - name: Generate SHA-256 checksums
        run: |
          cd dist/release
          sha256sum * > sha256sums.txt
          echo "=== sha256sums.txt ===" && cat sha256sums.txt
      - name: Sign with Sigstore (keyless, no stored private key)
        uses: sigstore/gh-action-sigstore-python@v3
        with:
          inputs: >-
            dist/release/wg-automate-linux-x86_64
            dist/release/wg-automate-macos-arm64
            dist/release/wg-automate-windows-x86_64.exe
      - name: Upload to GitHub Release
        uses: softprops/action-gh-release@v2
        with:
          files: |
            dist/release/wg-automate-linux-x86_64
            dist/release/wg-automate-macos-arm64
            dist/release/wg-automate-windows-x86_64.exe
            dist/release/sha256sums.txt
            dist/release/*.sigstore.json
```
