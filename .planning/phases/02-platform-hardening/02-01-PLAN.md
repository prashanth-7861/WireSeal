Create `src/wg_automate/platform/` directory with `__init__.py`.

**exceptions.py** -- Define the platform exception hierarchy:
- `PlatformError(Exception)` -- base for all platform errors
- `PrivilegeError(PlatformError)` -- not root/admin. Message must match locked format: "wg-automate requires root/Administrator privileges. Re-run with: sudo wg-automate" (Linux/macOS) or "Re-run from an elevated command prompt (Run as Administrator)" (Windows)
- `UnsupportedPlatformError(PlatformError)` -- unsupported OS
- `PrerequisiteError(PlatformError)` -- missing tools. Must include exact install command per locked decision
- `FirewallValidationError(PlatformError)` -- generated rules don't match deny-by-default template (FW-03)
- `SetupError(PlatformError)` -- generic setup step failure

**progress.py** -- Per locked decision, progress format is `[N/TOTAL] Description... done`. Create a `Progress` class:
- `__init__(self, total: int)` -- set total steps, current=0
- `step(self, description: str)` -- increment current, print `[{current}/{total}] {description}...` with `end=" ", flush=True`
- `done(self)` -- print `"done"`
- `fail(self, error: str)` -- print `"FAILED"` then on next line `"  Error: {error}"` then `"Setup failed at step {current}/{total}. Steps 1-{current-1} completed successfully. Fix the error above and re-run."`

Also add a module-level `validate_firewall_rules(generated: str, template: str) -> None` function in a new file or in `base.py` (put it in `base.py` since it's used by all adapters). This function:
1. Normalizes both strings: strip each line, remove blank lines, remove comment lines starting with `#`
2. Compares normalized versions
3. Raises `FirewallValidationError` if they differ
This satisfies FW-03.

**__init__.py** -- Export: `get_adapter`, `AbstractPlatformAdapter`, `Progress`, and all exception classes.

**base.py** -- Create `AbstractPlatformAdapter(ABC)` with these `@abstractmethod` methods (all raise `NotImplementedError` in the abstract body for documentation, but `@abstractmethod` enforces at instantiation):

1. `check_privileges(self) -> None` -- Raise PrivilegeError if not root/admin. Per locked decision: called at startup, BEFORE vault interaction.
2. `check_prerequisites(self) -> list[str]` -- Return list of missing tools. Raise PrerequisiteError with exact install commands if any missing. Per locked decision: "Run: apt install wireguard nftables" format.
3. `install_wireguard(self) -> None` -- Install WireGuard via platform package manager.
4. `deploy_config(self, config_content: str, interface: str = "wg0") -> Path` -- Write WireGuard config to platform-canonical path with correct permissions (delegates to `security.permissions.set_file_permissions`). Returns the path written. Uses `security.atomic.atomic_write`.
5. `apply_firewall_rules(self, wg_port: int, wg_interface: str, subnet: str) -> None` -- Apply deny-by-default + rate-limited UDP + NAT masquerade. Per locked decision: applied separately, NOT via PostUp/PostDown.
6. `remove_firewall_rules(self, wg_interface: str) -> None` -- Remove WireGuard firewall rules.
7. `enable_ip_forwarding(self) -> None` -- Enable IPv4 forwarding persistently.
8. `enable_tunnel_service(self, interface: str = "wg0") -> None` -- Enable and start the tunnel service.
9. `disable_tunnel_service(self, interface: str = "wg0") -> None` -- Disable and stop the tunnel service.
10. `setup_dns_updater(self, script_path: Path, interval_minutes: int = 5) -> None` -- Schedule DuckDNS updates as non-root user (HARD-04).
11. `get_config_path(self, interface: str = "wg0") -> Path` -- Return platform-canonical config path.
12. `detect_outbound_interface(self) -> str` -- Return name of default outbound network interface.

Also add a concrete helper method (not abstract):
- `validate_firewall_rules(self, generated: str, template: str) -> None` -- calls the module-level `validate_firewall_rules` from this file (FW-03). This is concrete so subclasses inherit it.

Add the `validate_firewall_rules` module-level function here (not in a separate file). Normalize both strings (strip lines, remove blanks and comments), compare, raise `FirewallValidationError` on mismatch.

**detect.py** -- Create:

1. `get_platform_info() -> dict` -- Returns `{"os": sys.platform, "version": platform.release(), "machine": platform.machine()}`. Used for logging and diagnostics.

2. `get_adapter() -> AbstractPlatformAdapter` -- Factory function:
   - `sys.platform == "linux"` -> lazy import `from .linux import LinuxAdapter; return LinuxAdapter()`
   - `sys.platform == "darwin"` -> lazy import `from .macos import MacOSAdapter; return MacOSAdapter()`
   - `sys.platform == "win32"` -> lazy import `from .windows import WindowsAdapter; return WindowsAdapter()`
   - else -> raise `UnsupportedPlatformError(f"Unsupported platform: {sys.platform}. wg-automate supports Linux, macOS, and Windows only.")`

Lazy imports are essential -- each platform module imports platform-specific stdlib (winreg, etc.) that would fail on other OSes.

Update `__init__.py` to export `get_adapter`, `AbstractPlatformAdapter`, `get_platform_info`.
