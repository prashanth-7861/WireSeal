**deploy_config(self, config_content: str, interface: str = "wg0") -> Path:**

**enable_ip_forwarding(self) -> None:**
- Write `/etc/sysctl.d/99-wireguard.conf` with content: `"net.ipv4.ip_forward = 1\n"` using `atomic_write`
- Apply immediately: `subprocess.run(["/sbin/sysctl", "-p", "/etc/sysctl.d/99-wireguard.conf"], shell=False, check=True, capture_output=True)`
- Idempotent: if file already exists with correct content, skip.

**apply_firewall_rules(self, wg_port: int, wg_interface: str, subnet: str) -> None:**
Per locked decisions: nftables ONLY, no iptables fallback. Idempotent: check before applying. Applied separately from PostUp/PostDown.

1. Detect outbound interface: `outbound = self.detect_outbound_interface()`
2. Generate nftables ruleset string using the template from research (table inet wg_filter + table ip wg_nat). Replace variables:
   - `$pub_iface` -> detected outbound interface
   - `$wg_iface` -> wg_interface param (always "wg0")
   - `$wg_port` -> wg_port param
   - subnet in masquerade rule
3. Build the expected template with same variable substitution (for FW-03 validation).
4. Call `self.validate_firewall_rules(generated_rules, template_rules)` -- raises FirewallValidationError if mismatch (FW-03).
5. Idempotency check: `subprocess.run(["nft", "list", "table", "inet", "wg_filter"], capture_output=True, shell=False)`. If returncode == 0, rules exist. Check if existing rules match by comparing `nft list table inet wg_filter` output structure. If match, skip. If different, flush and re-apply.
6. Write ruleset to `/etc/nftables.d/wireguard.nft` using `atomic_write`. Create `/etc/nftables.d/` if needed.
7. Apply: `subprocess.run(["nft", "-f", "/etc/nftables.d/wireguard.nft"], shell=False, check=True, capture_output=True, timeout=30)`

The nftables ruleset must contain (per FW-01 and FW-02):
- `policy drop` on input chain (deny-by-default)
- Rate limit: `limit rate over 5/second burst 10 packets drop` on WG port UDP
- `ct state { established, related } accept` for stateful tracking
- Forward chain: allow wg->outbound new, established/related
- NAT masquerade: `iifname $wg_iface oifname $pub_iface masquerade` (outbound interface only, NOT global -- FW-02)

**remove_firewall_rules(self, wg_interface: str) -> None:**
- `subprocess.run(["nft", "delete", "table", "inet", "wg_filter"], shell=False, capture_output=True)` (ignore error if table doesn't exist)
- `subprocess.run(["nft", "delete", "table", "ip", "wg_nat"], shell=False, capture_output=True)`
- Remove `/etc/nftables.d/wireguard.nft` if exists.

**enable_tunnel_service(self, interface: str = "wg0") -> None:**
- `subprocess.run(["systemctl", "enable", f"wg-quick@{interface}"], shell=False, check=True, capture_output=True)`
- `subprocess.run(["systemctl", "start", f"wg-quick@{interface}"], shell=False, check=True, capture_output=True)`

**disable_tunnel_service(self, interface: str = "wg0") -> None:**
- `subprocess.run(["systemctl", "stop", f"wg-quick@{interface}"], shell=False, capture_output=True)`
- `subprocess.run(["systemctl", "disable", f"wg-quick@{interface}"], shell=False, capture_output=True)`

**setup_dns_updater(self, script_path: Path, interval_minutes: int = 5) -> None:**
HARD-04: DuckDNS runs as non-root `wg-automate` system user.

1. Create system user if not exists:
   - Check: `subprocess.run(["id", "wg-automate"], capture_output=True, shell=False)`. If returncode == 0, skip.
   - Create: `subprocess.run(["adduser", "wg-automate", "--system", "--no-create-home", "--shell", "/usr/sbin/nologin", "--group", "--disabled-password"], shell=False, check=True, capture_output=True)`
2. Write cron entry to `/etc/cron.d/wg-automate`:
   ```
   # Managed by wg-automate -- DO NOT EDIT
   */{interval_minutes} * * * * wg-automate {script_path} update-dns --non-interactive
   ```
   Permissions: 0o644, owned by root (standard for cron.d files).
