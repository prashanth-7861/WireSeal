**check_privileges(self) -> None:**
- `ctypes.windll.shell32.IsUserAnAdmin()` returns 0 -> raise `PrivilegeError("wg-automate requires Administrator privileges. Re-run from an elevated command prompt (Run as Administrator).")`
- Per Claude's discretion: no auto-elevation via ShellExecuteEx runas. Just print the message and fail. This is consistent with the locked "no auto-escalation" decision.

**check_prerequisites(self) -> list[str]:**
- Check for WireGuard: `WG_EXE.exists()` or `shutil.which("wireguard")`
- Check for `netsh`: `shutil.which("netsh")` (should always exist but verify)
- Check for `winget`: `shutil.which("winget")`
- Check for `sc.exe`: `shutil.which("sc")` (service control)
- Check for `schtasks`: `shutil.which("schtasks")` (Task Scheduler)
- If WireGuard missing and winget available: raise `PrerequisiteError("Missing: WireGuard. Run: winget install --id WireGuard.WireGuard --silent")`
- If WireGuard missing and winget NOT available: raise `PrerequisiteError("Missing: WireGuard. Download from https://www.wireguard.com/install/ and install, then re-run.")`
- Return empty list if all present.

**install_wireguard(self) -> None:**
- If `WG_EXE.exists()`, return (idempotent).
- Run: `subprocess.run(["winget", "install", "--id", "WireGuard.WireGuard", "--silent", "--accept-package-agreements", "--accept-source-agreements"], shell=False, check=True, timeout=120, capture_output=True)`
- winget verifies package signatures automatically (per research).
- On CalledProcessError: raise `SetupError(f"WireGuard installation failed: {e.stderr.strip()}")`

**get_config_path(self, interface: str = "wg0") -> Path:**
- Return `WG_CONFIG_DIR / f"{interface}.conf"`
- Per locked decision: interface is always `wg0`, config is `wg0.conf`.

**deploy_config(self, config_content: str, interface: str = "wg0") -> Path:**
- Get path from `self.get_config_path(interface)`
- Create `WG_CONFIG_DIR` if not exists.
- Use `atomic_write(path, config_content.encode("utf-8"))` -- mode param ignored on Windows in atomic_write.
- Apply Windows ACL via `set_file_permissions(path)` from `security/permissions.py` (PLAT-06: SYSTEM + Administrators only, uses icacls).
- CRITICAL: Do NOT use `os.chmod()` on Windows for security (per PLAT-06 locked decision).
- Return the path.

**detect_outbound_interface(self) -> str:**
- Run PowerShell: `subprocess.run(["powershell", "-NoProfile", "-Command", "(Get-NetRoute -DestinationPrefix '0.0.0.0/0' | Sort-Object RouteMetric | Select-Object -First 1).InterfaceAlias"], capture_output=True, text=True, shell=False, check=True, timeout=30)`
- Return `stdout.strip()` (e.g., "Ethernet", "Wi-Fi")
- If empty: raise `SetupError("Cannot detect outbound network interface")`

**enable_ip_forwarding(self) -> None:**
Per research: requires registry write + reboot. Must warn user.

1. Read current value first:
   ```python
   key_path = r"SYSTEM\CurrentControlSet\Services\Tcpip\Parameters"
   with winreg.OpenKey(winreg.HKEY_LOCAL_MACHINE, key_path, 0, winreg.KEY_READ) as key:
       current_value, _ = winreg.QueryValueEx(key, "IPEnableRouter")
   ```
2. If `current_value == 1`: already enabled, skip (idempotent).
3. If `current_value == 0` or key doesn't exist:
   ```python
   with winreg.OpenKey(winreg.HKEY_LOCAL_MACHINE, key_path, 0, winreg.KEY_SET_VALUE) as key:
       winreg.SetValueEx(key, "IPEnableRouter", 0, winreg.REG_DWORD, 1)
   ```
4. Print warning to stderr: `"[!] IP routing enabled in registry. A system reboot is required for this to take effect."`
5. Write a sentinel file `WG_CONFIG_DIR / ".needs-reboot"` so re-run can detect pending reboot.
6. Note: The adapter does NOT force a reboot. The init command (Phase 4) will check for this sentinel and guide the user.

**apply_firewall_rules(self, wg_port: int, wg_interface: str, subnet: str) -> None:**
Per research: use netsh advfirewall. Target interface alias `wg0`, not GUID (per pitfall 3: GUID regenerates).

1. Detect outbound interface: `outbound = self.detect_outbound_interface()`
2. **Idempotency check:** query existing rules:
   ```python
   result = subprocess.run(
       ["netsh", "advfirewall", "firewall", "show", "rule", "name=wg-automate-wg0-in"],
       capture_output=True, text=True, shell=False
   )
   if result.returncode == 0 and "wg-automate-wg0-in" in result.stdout:
       return  # rules already exist
   ```
3. Build and validate firewall rules description for FW-03. Create a template string describing the expected rules, generate the actual rule commands, validate they match structurally.
4. Apply rules:
   ```python
   # Allow inbound WireGuard UDP on the configured port (FW-01: only WG traffic)
   subprocess.run(
       ["netsh", "advfirewall", "firewall", "add", "rule",
        "name=wg-automate-wg0-in", "protocol=UDP", "dir=in",
        f"localport={wg_port}", "action=allow", "profile=any", "enable=yes"],
       shell=False, check=True, capture_output=True
   )
   # Block all other inbound on WG interface (deny-by-default FW-01)
   subprocess.run(
       ["netsh", "advfirewall", "firewall", "add", "rule",
        "name=wg-automate-wg0-block", "dir=in",
        f"interface={wg_interface}", "action=block", "profile=any", "enable=yes"],
       shell=False, check=True, capture_output=True
   )
   ```
5. Set WG interface to Public profile (most restrictive):
   ```python
   subprocess.run(
       ["powershell", "-NoProfile", "-Command",
        f"Set-NetConnectionProfile -InterfaceAlias {wg_interface} -NetworkCategory Public"],
       shell=False, capture_output=True  # don't check -- interface may not exist yet
   )
   ```
6. For NAT (FW-02): Windows NAT via PowerShell `New-NetNat`:
   ```python
   # Enable NAT for the VPN subnet on the outbound interface only
   subprocess.run(
       ["powershell", "-NoProfile", "-Command",
        f"New-NetNat -Name 'wg-automate-nat' -InternalIPInterfaceAddressPrefix '{subnet}'"],
       shell=False, capture_output=True  # don't check -- may already exist
   )
   ```

**remove_firewall_rules(self, wg_interface: str) -> None:**
- `subprocess.run(["netsh", "advfirewall", "firewall", "delete", "rule", "name=wg-automate-wg0-in"], shell=False, capture_output=True)` (ignore error if not found)
- `subprocess.run(["netsh", "advfirewall", "firewall", "delete", "rule", "name=wg-automate-wg0-block"], shell=False, capture_output=True)`
- `subprocess.run(["powershell", "-NoProfile", "-Command", "Remove-NetNat -Name 'wg-automate-nat' -Confirm:$false"], shell=False, capture_output=True)`

**enable_tunnel_service(self, interface: str = "wg0") -> None:**
Per research: `wireguard.exe /installtunnelservice` creates the service. The Manager Service auto-encrypts `.conf` to `.conf.dpapi`.

1. Get config path: `config_path = self.get_config_path(interface)`
2. Verify config exists: if not, raise `SetupError(f"Config not found: {config_path}. Deploy config first.")`
3. Install tunnel service:
   ```python
   subprocess.run(
       [str(WG_EXE), "/installtunnelservice", str(config_path)],
       shell=False, check=True, capture_output=True, timeout=30
   )
   ```
   This creates service `WireGuardTunnel$wg0`. The Manager Service auto-encrypts the .conf to .conf.dpapi (DPAPI-bound to LocalSystem). The original .conf is deleted after encryption.
4. Configure auto-start: `subprocess.run(["sc.exe", "config", f"{WG_SERVICE_PREFIX}{interface}", "start=auto"], shell=False, check=True, capture_output=True)`
5. Start the service: `subprocess.run(["sc.exe", "start", f"{WG_SERVICE_PREFIX}{interface}"], shell=False, capture_output=True)` (don't check -- may already be running)

**disable_tunnel_service(self, interface: str = "wg0") -> None:**
1. Stop: `subprocess.run(["sc.exe", "stop", f"{WG_SERVICE_PREFIX}{interface}"], shell=False, capture_output=True)`
2. Uninstall: `subprocess.run([str(WG_EXE), "/uninstalltunnelservice", interface], shell=False, capture_output=True)`

**setup_dns_updater(self, script_path: Path, interval_minutes: int = 5) -> None:**
HARD-04: DuckDNS runs as a low-privilege local user via Task Scheduler.

1. Create low-privilege user if not exists:
   - Check: `subprocess.run(["net", "user", "wg-automate-dns"], capture_output=True, shell=False)`. If returncode == 0, skip.
   - Generate a random password (16 chars) for the service account using `secrets.token_urlsafe(16)`. This is only used to create the account; the scheduled task uses the `/ru` flag.
   - Create: `subprocess.run(["net", "user", "wg-automate-dns", password, "/add", "/expires:never", "/passwordchg:no", "/comment:wg-automate DNS update service account"], shell=False, check=True, capture_output=True)`
   - Deny interactive logon (least privilege): `subprocess.run(["net", "localgroup", "Users", "wg-automate-dns", "/delete"], shell=False, capture_output=True)` (remove from Users group)
2. Create scheduled task:
   ```python
   subprocess.run(
       ["schtasks", "/create", "/tn", "WgAutomateDNS",
        "/tr", f'"{script_path}" update-dns --non-interactive',
        "/sc", "minute", "/mo", str(interval_minutes),
        "/ru", "wg-automate-dns",
        "/rp", password,
        "/f"],  # /f = force overwrite if exists (idempotent)
       shell=False, check=True, capture_output=True
   )
   ```
3. Wipe the password from memory after use (best-effort via overwrite).
