**enable_ip_forwarding(self) -> None:**
Per research: `sysctl.conf` is unreliable on Sonoma/Sequoia. Use runtime `sysctl -w` plus a companion launchd plist for boot persistence.

1. Apply immediately: `subprocess.run(["/usr/sbin/sysctl", "-w", "net.inet.ip.forwarding=1"], shell=False, check=True, capture_output=True)`
2. Create a boot-persistence launchd plist at `/Library/LaunchDaemons/com.wg-automate.sysctl.plist` using `plistlib.dumps()`:
   ```python
   plist_data = {
       "Label": "com.wg-automate.sysctl",
       "ProgramArguments": ["/usr/sbin/sysctl", "-w", "net.inet.ip.forwarding=1"],
       "RunAtLoad": True,
       "LaunchOnlyOnce": True,
   }
   ```
   Write with `atomic_write`, then set ownership root:wheel (`chown root:wheel`) and permissions 644 (`chmod 644`).
3. Load it: `subprocess.run(["launchctl", "bootstrap", "system", str(plist_path)], shell=False, capture_output=True)` (ignore error if already loaded).

**apply_firewall_rules(self, wg_port: int, wg_interface: str, subnet: str) -> None:**
Per research: use pfctl anchors, NEVER edit /etc/pf.conf. Anchor: `com.apple/wireguard`.

1. Detect outbound interface: `outbound = self.detect_outbound_interface()`
2. Generate PF rules string (order matters for macOS pf -- last match wins by default, but with `quick` keyword, first match wins):
   ```
   # wg-automate managed rules -- DO NOT EDIT
   pass in quick on {outbound} proto udp from any to any port {wg_port} keep state
   pass out quick on {outbound} from {subnet} to any nat-to ({outbound})
   block drop in on {outbound} all
   ```
   NOTE: The `nat-to` syntax is used for PF NAT on modern macOS. The `nat on` syntax is for the global NAT section which anchors don't support directly. Use `pass ... nat-to` for inline NAT in anchors.

   Actually, per the research, macOS pfctl anchors support separate NAT rules. Also add rate limiting via `max-pkt-rate` state option (PF on macOS 10.7+ supports it):
   ```
   nat on {outbound} from {subnet} to any -> ({outbound})
   table <wg_bruteforce> persist
   block drop in quick on {outbound} from <wg_bruteforce>
   pass in quick on {outbound} proto udp from any to any port {wg_port} \
       keep state (max-pkt-rate 5/1, overload <wg_bruteforce> flush global)
   block drop in on {outbound} all
   ```
   This limits new UDP connections to 5/second (burst tracked by the overload table `wg_bruteforce`); hosts exceeding the rate are blocked via the table.

3. Build template with same values for FW-03 validation: `self.validate_firewall_rules(generated, template)`.
4. Idempotency: check if anchor already has rules: `subprocess.run(["pfctl", "-a", "com.apple/wireguard", "-sr"], capture_output=True, text=True, shell=False)`. If output matches expected rules, skip.
5. Apply rules to anchor: `subprocess.run(["pfctl", "-a", "com.apple/wireguard", "-f", "-"], input=rules.encode(), shell=False, check=True, capture_output=True)`
6. Enable pf if not already enabled: `subprocess.run(["pfctl", "-e"], shell=False, capture_output=True)` (ignore "already enabled" error).
7. Save the PF token if `-E` flag is used (for cleanup). Write token to `/var/run/wg-automate/pf_token`.

**remove_firewall_rules(self, wg_interface: str) -> None:**
- Flush anchor: `subprocess.run(["pfctl", "-a", "com.apple/wireguard", "-F", "all"], shell=False, capture_output=True)`
- If token file exists at `/var/run/wg-automate/pf_token`: read token, run `pfctl -X {token}`, remove token file.

**enable_tunnel_service(self, interface: str = "wg0") -> None:**
Create and load a launchd daemon plist.

1. Get brew prefix and wg-quick path: `wg_quick = Path(self._get_brew_prefix()) / "bin" / "wg-quick"`
2. Get config path: `config_path = self.get_config_path(interface)`
3. Build plist using `plistlib`:
   ```python
   plist_data = {
       "Label": f"com.wg-automate.{interface}",
       "ProgramArguments": [str(wg_quick), "up", str(config_path)],
       "KeepAlive": True,
       "RunAtLoad": True,
       "LaunchOnlyOnce": True,
       "StandardErrorPath": "/var/log/wg-automate.err",
       "EnvironmentVariables": {
           "PATH": f"{self._get_brew_prefix()}/sbin:{self._get_brew_prefix()}/bin:/usr/bin:/bin:/usr/sbin:/sbin"
       },
   }
   ```
4. Write to `/Library/LaunchDaemons/com.wg-automate.{interface}.plist` using `atomic_write`.
5. Set ownership and permissions: `chown root:wheel`, `chmod 644`.
6. Enable and bootstrap:
   - `subprocess.run(["launchctl", "enable", f"system/com.wg-automate.{interface}"], shell=False, check=True, capture_output=True)`
   - `subprocess.run(["launchctl", "bootstrap", "system", str(plist_path)], shell=False, check=True, capture_output=True)`

**disable_tunnel_service(self, interface: str = "wg0") -> None:**
- `subprocess.run(["launchctl", "bootout", f"system/com.wg-automate.{interface}"], shell=False, capture_output=True)`
- `subprocess.run(["launchctl", "disable", f"system/com.wg-automate.{interface}"], shell=False, capture_output=True)`
- Also run `wg-quick down {interface}` to clean up the utun interface.

**setup_dns_updater(self, script_path: Path, interval_minutes: int = 5) -> None:**
HARD-04: DuckDNS runs as non-admin user via launchd `UserName` key.

1. Create a non-admin user `wg-automate` if not exists:
   - Check: `subprocess.run(["dscl", ".", "-read", "/Users/wg-automate"], capture_output=True, shell=False)`. If returncode == 0, skip.
   - Create system user via `dscl`:
     ```
     dscl . -create /Users/wg-automate
     dscl . -create /Users/wg-automate UserShell /usr/bin/false
     dscl . -create /Users/wg-automate UniqueID 399  (find available UID < 500)
     dscl . -create /Users/wg-automate PrimaryGroupID 399
     dscl . -create /Users/wg-automate NFSHomeDirectory /var/empty
     ```
   - Find an available UID: iterate from 400 down to 300 checking `dscl . -read /Users/uid_{n}`.
2. Write launchd plist to `/Library/LaunchDaemons/com.wg-automate.dns.plist`:
   ```python
   plist_data = {
       "Label": "com.wg-automate.dns",
       "ProgramArguments": [str(script_path), "update-dns", "--non-interactive"],
       "StartInterval": interval_minutes * 60,
       "UserName": "wg-automate",
       "StandardErrorPath": "/var/log/wg-automate-dns.err",
   }
   ```
3. Load with `launchctl bootstrap system`.
