**src/wg_automate/security/secrets_wipe.py:**

Implement two functions:

`wipe_bytes(data: bytearray) -> None`:
- Overwrites every byte with 0x00 (zero pass)
- Overwrites every byte with random values via `os.urandom(len(data))` (random pass)
- Overwrites every byte with 0x00 again (final zero pass)
- Uses `ctypes.memmove(ctypes.c_char_p(id(data) + offset), source, len(data))` approach OR simple index-based loop. Prefer the simple loop approach for clarity since bytearray supports item assignment.
- No-op if data has length 0

`wipe_string(s: str) -> None`:
- Best-effort wipe of a Python string's internal buffer via ctypes
- Use `ctypes.memset(id(s) + offset, 0, len(s))` where offset accounts for CPython's PyUnicodeObject header
- The offset for CPython compact ASCII strings is typically 48 bytes (sys.getsizeof(s) - len(s) gives the header size)
- Wrap in try/except and silently pass on failure -- this is best-effort since strings are immutable and the CPython internals may vary
- Add a comment: `# Best-effort: CPython implementation detail, not guaranteed`

**src/wg_automate/security/secret_types.py:**

Implement `SecretBytes` class:

```
class SecretBytes:
    """Mutable secret container that prevents accidental exposure and wipes memory on release."""
```

Constructor `__init__(self, data: bytearray | bytes)`:
- If `data` is `bytes`, convert to `bytearray` immediately (SEC-04: never hold in immutable type)
- Store as `self._data: bytearray`
- Call `self._mlock()` to pin memory (best-effort, SEC-02)
- Set `self._wiped = False`

Memory locking `_mlock(self)`:
- On Linux/macOS: `ctypes.CDLL("libc.so.6" or "libSystem.B.dylib").mlock(address, length)`
  - Get buffer address via `ctypes.addressof(ctypes.c_char.from_buffer(self._data))`
  - Wrap in try/except, pass on failure (best-effort)
- On Windows: `ctypes.windll.kernel32.VirtualLock(address, length)`
  - Same try/except pattern
- Use `sys.platform` to select the right approach

Memory unlocking `_munlock(self)`:
- Reverse of mlock: `munlock()` on Unix, `VirtualUnlock()` on Windows
- Called in `wipe()` after zeroing

`wipe(self) -> None`:
- Call `secrets_wipe.wipe_bytes(self._data)` (SEC-03)
- Call `self._munlock()`
- Set `self._wiped = True`

`__del__(self)`:
- Safety net: call `self.wipe()` if not already wiped

`__repr__(self) -> str`: return `"SecretBytes(***)"` -- NEVER expose content (SEC-01)

`__str__(self) -> str`: return `"SecretBytes(***)"` -- NEVER expose content (SEC-01)

`__eq__(self, other) -> bool`:
- Use `hmac.compare_digest(self._data, other._data)` for constant-time comparison (SEC-01)
- Return `NotImplemented` if other is not SecretBytes

`__hash__(self)`: raise `TypeError("SecretBytes is not hashable")` -- prevent use as dict key where content could leak via hash collisions (SEC-01)

`__getstate__(self)`: raise `TypeError("SecretBytes cannot be pickled")` -- prevent serialization (SEC-01)

`__len__(self) -> int`: return `len(self._data)`

`__bytes__(self) -> bytes`: return `bytes(self._data)` -- explicit conversion when needed (caller responsible for wiping)

`__enter__(self)`: return self

`__exit__(self, *args)`: call `self.wipe()` -- context manager for automatic cleanup (SEC-05)

`to_bytearray(self) -> bytearray`: return a COPY as bytearray (caller responsible for wiping the copy)

`expose_secret(self) -> bytearray`: return reference to internal `self._data` -- clearly named dangerous method for when raw access is truly needed

Property `is_wiped -> bool`: return `self._wiped`
