**src/wg_automate/security/atomic.py:**

Extract the `_atomic_write` function into a standalone shared module so both `vault.py` and `config_builder.py` can import it without circular dependencies.

`atomic_write(path: Path, data: bytes, mode: int = 0o600) -> None`:
- `tempfile.mkstemp(dir=path.parent, prefix='.tmp_wga_')`
- Write data via `os.write(fd, data)`
- `os.fsync(fd)`
- `os.close(fd)`
- Unix: `os.chmod(tmp_path, mode)` BEFORE rename (file never world-readable)
- `os.replace(tmp_path, str(path))` (cross-platform atomic, works on Windows unlike os.rename)
- Unix: fsync parent directory (`os.fsync(os.open(str(path.parent), os.O_RDONLY))`)
- Windows: skip parent fsync (NTFS journaling)
- Wrap in try/except BaseException: clean up tmp file on any failure (delete tmp in finally)

After creating `atomic.py`, update `vault.py` (from plan 01-02) to import `atomic_write` from `wg_automate.security.atomic` instead of using its inline `_atomic_write`. Remove the inline copy from vault.py.

**src/wg_automate/security/validator.py:**

All validation functions raise `ValueError` with precise error messages including field name, invalid character, and position (locked decision: compiler-quality error messages).

`validate_client_name(name: str) -> None` (CONFIG-06):
- Pattern: `re.compile(r'^[a-zA-Z0-9-]{1,32}$')`
- Empty name: `ValueError("Client name cannot be empty")`
- Too long: `ValueError(f"Client name '{name}' exceeds 32-character limit ({len(name)} chars)")`
- Invalid char: iterate to find first offending char and position: `ValueError(f"Client name '{name}' contains invalid character '{ch}' at position {i}")`

`validate_wg_key(key: str, field_name: str) -> None` (CONFIG-02):
- Length check: `if len(key) != 44: raise ValueError(f"Field '{field_name}': expected 44-character base64 key, got {len(key)} characters")`
- Decode with `base64.b64decode(key, validate=True)` -- catch `binascii.Error`, raise `ValueError(f"Field '{field_name}': invalid base64 encoding -- {e}") from None`
- Byte length: `if len(decoded) != 32: raise ValueError(f"Field '{field_name}': key decodes to {len(decoded)} bytes, expected 32")`

`validate_port(port: int, field_name: str = "port") -> None` (CONFIG-02):
- `if not (1024 <= port <= 65535): raise ValueError(f"Field '{field_name}': port {port} outside valid range 1024-65535")`

`validate_subnet(subnet: str, field_name: str = "subnet") -> None`:
- Parse with `ipaddress.ip_network(subnet, strict=True)` -- catch ValueError, raise with field name
- `if not net.is_private: raise ValueError(f"Field '{field_name}': subnet '{subnet}' is not an RFC 1918 private range")`

`validate_ip(ip: str, subnet: str, field_name: str = "ip") -> None`:
- Parse with `ipaddress.ip_address(ip)` -- catch ValueError
- Check `ip_address(ip) in ip_network(subnet, strict=False)` -- raise if not in subnet
- Check `ip_address(ip).is_private` -- raise if not RFC 1918

`validate_no_injection(value: str, field_name: str) -> None` (CONFIG-02):
- Pattern: `re.compile(r'[\[\]=\n\r]')` -- characters that break WireGuard INI format
- On match: `ValueError(f"Field '{field_name}': contains INI-injection character '{repr(char)}' at position {pos}")`

`validate_allowed_ips(allowed_ips: str, field_name: str = "AllowedIPs") -> None`:
- Split on `,`, strip whitespace
- Each entry must be valid CIDR: `ipaddress.ip_network(entry, strict=False)`
- Raise ValueError with field name and invalid entry

`validate_server_config(config: dict) -> None`:
- Orchestrates: validate server private key, public key, port, subnet
- For each client: validate client name, public key, PSK, IP (within subnet), no injection on name

`validate_client_config(config: dict) -> None`:
- Orchestrates: validate client private key, PSK, IP, DNS server, server public key, endpoint

**src/wg_automate/security/integrity.py:**

`compute_config_hash(config_path: Path) -> str` (CONFIG-04):
- `return hashlib.sha256(config_path.read_bytes()).hexdigest()`

`verify_config_integrity(config_path: Path, stored_hash: str) -> bool`:
- Compute current hash
- Compare using `hmac.compare_digest(actual, stored)` (constant-time)
- Return True if match, False if mismatch
- Note: the CALLER handles the security alert message (locked decision: "SECURITY ALERT: Config file tampered -- aborting. Do not reload WireGuard.") and sys.exit -- this function just returns a boolean

`store_hash_in_state(state_dict: dict, config_name: str, hash_value: str) -> None`:
- `state_dict.setdefault('integrity', {})[config_name] = hash_value`
- Also store timestamp: `state_dict['integrity'][f'{config_name}_verified'] = datetime.utcnow().isoformat() + 'Z'`

**src/wg_automate/security/permissions.py:**

`set_file_permissions(path: Path, mode: int = 0o600) -> None` (CONFIG-03):
- Unix: `os.chmod(path, mode)`
- Windows: use `subprocess.run(['icacls', str(path), '/inheritance:r', '/grant:r', 'SYSTEM:(R,W)', 'Administrators:(R,W)'], check=True, capture_output=True)` to set SYSTEM+Administrators only
- Wrap Windows call in try/except; if icacls fails, try pywin32 if available, else log warning

`set_dir_permissions(path: Path, mode: int = 0o700) -> None` (VAULT-02):
- Unix: `os.chmod(path, mode)`
- Windows: similar icacls call with full control for SYSTEM+Administrators

`check_file_permissions(path: Path) -> bool`:
- Unix: `stat.S_IMODE(path.stat().st_mode) == 0o600`
- Windows: best-effort check via icacls output parsing
- Returns True if permissions are correct
