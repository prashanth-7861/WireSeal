**Edit `src/wireseal/api.py`** ‚Äî add `/api/changelog` handler:
```python
def _h_changelog(_req, _groups):
    import sys, os
    base = getattr(sys, "_MEIPASS", os.path.dirname(os.path.abspath(__file__)))
    # _MEIPASS is PyInstaller runtime dir; fallback for dev mode
    candidates = [
        Path(base) / "CHANGELOG.md",
        Path(base).parent.parent / "CHANGELOG.md",  # dev from src/wireseal/
    ]
    for p in candidates:
        if p.is_file():
            return {"markdown": p.read_text(encoding="utf-8")}
    return {"markdown": "# Changelog\n\nChangelog not bundled."}
```
Register in `_ROUTES`.

## Phase 3 ‚Äî Client/Server mode switch (P0)

**Goal:** "Switch to Client" actually re-initializes the vault in client mode (or refuses with clear message).

### Decision needed first
Mode is set at vault init and baked into the encrypted vault data. Three options:

- **Option A ‚Äî Dual vaults:** Server vault at `vault.enc`, client vault at `vault-client.enc`. Switching just changes which file is active. (Clean but doubles complexity.)
- **Option B ‚Äî Export/import:** Disable mode switch post-init. Button becomes "Create client profile" which launches a wizard to generate a new vault file. User must explicitly unlock one or the other.
- **Option C ‚Äî Hide the switch on server-mode vaults.** Simplest. Re-initialize requires Fresh Start. Reword button to "Client mode requires a separate vault ‚Äî use Fresh Start to re-initialize".

**Recommendation: Option C** for v0.7.14 (smallest change, honest UX). Option B as follow-up.

### Implementation (Option C)
**Edit `Dashboard/src/app/components/Layout.tsx:933`**:
- Remove the mode-switch onClick behavior.
- Replace with tooltip-only display: `"Mode: server ‚Äî re-initialize via Fresh Start to change"`.
- Or: gate the button behind a modal that explains the Fresh Start flow.

**Edit `Dashboard/src/app/context/AppModeContext.tsx`**:
- Remove `clearMode()` export if no longer called.
- Mode becomes read-only, derived from `api.status().mode`, not localStorage.

**Edit `Dashboard/src/app/App.tsx`** (or wherever probe runs):
- On unlock, always read mode from `/api/status` response. Never trust localStorage.
- Clear any stale `localStorage.wireseal_mode` key on startup.

**Files touched:** `Dashboard/src/app/components/Layout.tsx`, `Dashboard/src/app/context/AppModeContext.tsx`, `Dashboard/src/app/App.tsx`.
